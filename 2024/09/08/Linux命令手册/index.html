命令合集

1. 基础命令

  • echo: 用于打印消息或输出变量的值。

  • cd: 更改当前目录。

  • pwd: 显示当前工作目录。

  • ls: 列出目录中的文件和子目录。

  • touch: 创建空文件。

  • cp: 复制文件或目录。

  • mv: 移动或重命名文件/目录。

  • rm: 删除文件或目录。

2. 条件语句

条件语句在 Linux 脚本编程中用于根据特定条件执行不同的操作。条件语句通常用于检查文件、变量值或其他条件，并基于这些条件控制程序的流程。主要的条件语句包括 if-else、case 和 test。以下是详细的解释和示例：

  • **-f "$file"**：检查指定的路径是否存在且是一个文件。
  • **"$file"**：变量引用，双引号防止路径中有空格导致的问题。

8. 文件处理

创建文件

  • **touch**：创建一个空文件或更新现有文件的时间戳。

    • 如果 filename.txt 已存在，touch 会更新文件的最后修改时间。
    • 如果 filename.txt 不存在，则会创建一个空文件。

  • **echo**：创建一个文件并写入内容。

    • 使用 > 重定向符将文本 "Hello, World!" 写入 hello.txt 文件。如果文件已存在，会被覆盖。

读取文件

  • **cat**：显示文件内容。

    • cat 命令会将整个文件的内容输出到标准输出（通常是终端）。

  • more 和 **less**：分页显示文件内容。

    • **more**：逐页显示文件内容，按空格键翻页，按 q 退出。
    • **less**：比 more 更强大，支持前后翻页，按 q 退出。

  • **head**：显示文件的开头部分。

    • -n 选项指定显示的行数，默认显示前10行。

  • **tail**：显示文件的结尾部分。

    • -n 选项指定显示的行数，默认显示最后10行。
    • **tail -f filename.txt**：实时显示文件新增的内容，用于监控日志文件。

写入文件

  • **>**：将输出重定向到文件，覆盖文件内容。

  • **>>**：将输出追加到文件末尾。

  • **tee**：将输出写入文件并显示在终端。

    • tee 命令会将输出内容写入 filename.txt 文件，并同时输出到标准输出。

查找和处理文件

  • **find**：查找符合条件的文件。

    • 查找 /path/to/dir 目录下所有 .txt 文件。

  • **grep**：在文件中查找匹配的文本。

    • 查找 filename.txt 中包含 "search_term" 的行。

  • **sed**：流编辑器，用于查找和替换文本。

    • 将 filename.txt 中的所有 old 替换为 new。

  • **awk**：文本处理工具，用于格式化和处理数据。

    • 打印 filename.txt 中每一行的第一个字段。

修改文件

  • **mv**：移动或重命名文件。

    • 将 oldname.txt 重命名为 newname.txt。
    • 也可以将文件移动到不同目录。

  • **cp**：复制文件。

    • 复制 source.txt 文件到 destination.txt。

  • **rm**：删除文件。

    • 删除 filename.txt 文件。

  • **chmod**：修改文件权限。

    • 修改 filename.txt 的权限为 rwxr-xr-x。

处理文本文件

  • **sort**：对文件内容进行排序。

    • 对 filename.txt 的内容进行升序排序。

  • **uniq**：去除重复行。

    • uniq 通常与 sort 命令结合使用，用于去除重复行。

  • **cut**：从文件中提取特定字段。

    • 提取 filename.txt 中以逗号分隔的第一列数据。

文件权限

  • **chmod**：修改文件权限。

    • 将 filename.txt 的权限设置为 rw-r--r--（用户可读写，组用户和其他用户可读）。

  • **chown**：修改文件的所有者和组。

    • 将 filename.txt 的所有者更改为 user，组更改为 group。

文件比较

  • **diff**：比较两个文件的差异。

    • 显示 file1.txt 和 file2.txt 之间的差异。

  • **cmp**：逐字节比较两个文件。

    • 比较两个文件的内容，显示第一个不同的字节位置。

归档和压缩

  • **tar**：创建和解压归档文件。

    • 创建一个名为 archive.tar 的归档文件，包含 file1.txt 和 file2.txt。
    • 解压 archive.tar 归档文件。

  • gzip 和 **gunzip**：压缩和解压文件。

    • 压缩 filename.txt 文件，生成 filename.txt.gz。
    • 解压 filename.txt.gz 文件，恢复为 filename.txt。

11. 权限

在 Linux 系统中，文件和目录的权限决定了哪些用户或用户组可以对文件进行读、写、执行操作。每个文件和目录的权限由三组表示：

  1. 用户权限（owner）：文件的所有者。
  2. 用户组权限（group）：文件所属的用户组。
  3. 其他用户权限（others）：系统中其他用户的权限。

查看权限：

使用 ls -l 命令可以查看文件的权限：

输出示例：

  • 第一个字符表示文件类型（- 表示普通文件，d 表示目录）。
  • 后面九个字符分别表示用户、组和其他人的权限。
    • r：可读（read）。
    • w：可写（write）。
    • x：可执行（execute）。
    • 没有权限时用 - 表示。

如上面的例子：

  • rwx：用户（owner）可以读、写、执行。
  • r-x：组用户可以读和执行，但不能写。
  • r--：其他用户只能读。

修改权限：

使用 chmod 命令修改文件或目录的权限。

  1. 数字模式：

权限用三个数字表示，每个数字为权限组合值的总和：

  • 读（r）：4
  • 写（w）：2
  • 执行（x）：1

组合值：

  • 7 = 读 + 写 + 执行
  • 6 = 读 + 写
  • 5 = 读 + 执行
  • 4 = 读

  • u：用户（owner）
  • g：组用户
  • o：其他用户
  • **+**：添加权限
  • **-**：移除权限
  • **=**：设置权限

12. 进程控制

Linux 是一个多任务操作系统，进程控制是管理和操作系统中运行的程序或任务的重要功能。通过管理进程，用户可以查看、终止、暂停和恢复程序的执行。

查看进程：

使用 ps 命令查看系统中当前运行的进程。常用选项包括：

  • **ps**：显示当前进程信息。
  • **aux**：显示所有用户的所有进程以及详细信息。

示例输出：

  • PID：进程 ID。
  • %CPU：CPU 使用率。
  • %MEM：内存使用率。
  • COMMAND：正在执行的命令。

后台运行进程

使用 & 将进程放在后台执行：

  • **&**：将脚本放入后台执行，终端不会等待脚本执行完毕。

停止和恢复进程

  • **Ctrl + Z**：暂停前台运行的进程。
  • **bg**：将暂停的进程放入后台继续运行。
  • **fg**：将后台进程恢复到前台。

终止进程

使用 kill 命令可以终止进程。

  • **<PID>**：进程的 ID，通过 ps 命令获得。

  2. **htop**：功能更强大的进程监控工具，需要单独安装。

  3. **jobs**：查看当前会话中的后台作业。

使用示例：

grep - 文本搜索工具

  1. 查找文件中的文本

    • 功能：在 filename.txt 中查找包含 search_term 的行，并显示这些行。
    • 常用选项：
      • -i：忽略大小写。
      • -v：显示不包含 search_term 的行。
      • -r 或 -R：递归地在目录中查找文件。
      • -n：显示匹配行的行号。
      • -H：显示匹配的文件名。
    • 功能：在 /var/log/syslog 文件中查找包含 error（忽略大小写）的行。

  2. 查找进程相关信息

    • 功能：列出所有进程，然后通过 grep 过滤出包含 process_name 的进程信息。
    • 常用选项：
      • -e：显示所有进程。
      • -f：显示详细信息。
      • aux：结合 a（所有用户的进程）、u（以用户格式显示）和 x（显示没有控制终端的进程）。
    • 功能：列出所有进程，并显示包含 apache 的行，用于查找与 Apache 相关的进程。

  3. 显示指定目录下所有文件中包含特定内容的行

    • 功能：递归搜索 /path/to/dir 目录及其子目录中的所有文件，查找包含 search_term 的行。

top - 实时系统监控工具

  1. 基本使用

    • 功能：显示系统上当前运行的进程及其资源使用情况，实时更新。
    • 常用功能：
      • **P**：按 CPU 使用率排序。
      • **M**：按内存使用情况排序。
      • **q**：退出 top。

  2. 指定更新间隔

    • 功能：每隔 5 秒更新一次显示。

  3. 显示指定用户的进程

    • 功能：仅显示 username 用户的进程。

  4. 自定义显示

    • 功能：以批处理模式运行 top，输出一次快照并退出（适合将输出重定向到文件）。

ps - 显示进程状态

  1. 显示所有进程

    • 功能：显示所有用户的所有进程，提供详细信息，包括 CPU 和内存使用情况。
    • 常用选项：
      • a：显示所有用户的进程。
      • u：以用户格式显示。
      • x：显示没有控制终端的进程。

  2. 显示特定用户的进程

    • 功能：仅显示 username 用户的进程。

  3. 显示特定进程的信息

    • 功能：显示进程 ID 为 PID 的进程信息。

  4. 显示树状进程结构

    • 功能：以树状结构显示进程及其子进程。

结合使用

  1. 查找占用高 CPU 的进程

    • 功能：运行 top 命令并筛选出 CPU 使用情况的行。

  2. 检查特定进程的资源使用情况

    • 功能：查找 process_name 相关的进程，并使用 awk 提取用户、PID、CPU 使用率、内存使用率和命令名。

  3. 实时监控进程状态

    • 功能：实时监控与 process_name 相关的进程，pgrep 用于查找进程 ID，并将其传递给 top 命令。
